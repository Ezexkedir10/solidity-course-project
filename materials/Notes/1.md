Solidity Introduction 

 Smart Contracts
    Solidity is a programming language used to write smart contracts on blockchain platforms such as Ethereum. It is similar to JavaScript in syntax and is mainly used for decentralized applications (DApps).

Smart Contracts Overview
    A smart contract is a program stored on the blockchain that automatically executes when conditions are met.

Features of Smart Contracts
    Decentralized
    Immutable after deployment
    Transparent
    Automatic execution
    Secure through blockchain consensus
Example
pragma solidity ^0.8.0;

contract HelloWorld {
    string public message = "Hello Blockchain!";
}

Value Types
    Value types store data directly and pass copies during assignments.

Common Value Types
    Type	Example
    bool	true / false
    uint	positive integers
    int	positive & negative integers
    address	wallet addresses
    bytes	raw byte data
Example
bool public isActive = true;
uint public age = 20;
int public score = -5;

Functions and Value Types
    Functions perform actions inside contracts.

Function Syntax
function add(uint a, uint b) public pure returns(uint) {
    return a + b;
}

Keywords
Keyword	Meaning
public	callable from anywhere
private	callable only inside contract
view	reads blockchain data
pure	does not read/write blockchain
Storage Variables

Storage variables are permanently stored on the blockchain.

Example
uint public count = 0;

Changing storage variables costs gas because data is written to the blockchain.

Functions

Functions can:

receive parameters
return values
modify blockchain state
Example
contract Counter {
    uint public count;

    function increment() public {
        count += 1;
    }
}
